Usage

Use this markup to create a toggle. [data-xt-toggle] must be initialized by javascript.
Instead of using html arguments you can initialize toggle by javascript using the same options syntax.

Defaults

Here are the defaults options for toggle.

  
XtToggle.defaults = {
  "elements": ":scope > a, :scope > button",
  "targets": ":scope > [class^=\"toggle-\"], :scope > [class*=\" toggle-\"]",
  "class": "active",
  "instant": {"elements": true},
  "on": "click",
  "toggle": true,
  "min": 0,
  "max": 1,
  "aria": true
};

Subcomponents

Drop

Usage

Drop is a special type of toggle. Use this markup to create a drop. [data-xt-drop] must be initialized by javascript.

Instead of using html arguments you can initialize drop by javascript using the same options syntax.
Defaults

Here are the defaults options for drop.

  
XtDrop.defaults = {
  "targets": ":scope > .drop",
  "controls": ":scope > a, :scope > button",
  "class": "active",
  "instant": {"elements": true},
  "on": "click",
  "toggle": true,
  "min": 0,
  "max": 1,
  "closeOutside": "body",
  "aria": true
};

Overlay

Usage

Overlay is a special type of toggle. Use this markup to create a overlay. [data-xt-overlay] must be initialized by javascript.

Instead of using html arguments you can initialize overlay by javascript using the same options syntax.
Defaults

Here are the defaults options for overlay.

  
XtOverlay.defaults = {
  "elements": ":scope > a, :scope > button",
  "targets": ":scope > .overlay-outer",
  "class": "active",
  "instant": {"elements": true},
  "on": "click",
  "toggle": true,
  "min": 0,
  "max": 1,
  "appendTo": "body",
  "backdrop": "targets",
  "classHtml": "xt-overlay",
  "closeInside": ".overlay-dismiss, :scope > .xt-backdrop, :scope .overlay-inner > .btn-close",
  "scrollbar": true,
  "aria": true
};

Mode

Preferred mode is multiple, but unique is useful when triggering targets outside the scope, like overlay where the targets are moved to body.

For toggle utils classes read the documentation of toggle utils.

Multiple

You can create a "mutiple" mode by assigning [data-xt-toggle] to a scope element, elements and targets must be inside the scope. Check toggle's defaults for the default elements and targets.

With multiple mode you can't specify targets with #id.
With multiple targets, the ones activated is the one with the same index of the toggled element. Nested targets are filtered out

Min and max

Inside a manager you can specify min or max activations. Default is "min": 0 and "max": 1.

Use "max": "Infinity" for no limits.

The required min activation is fulfilled on initialization

Auto

You can set automatic activation with "auto": Milliseconds and "autoPause": Milliseconds on user interaction. At the end it automatically loops.

Use "autoPause": "stop" for stopping on interaction.

Use "autoAlways": true to execute auto always, by default it only execute auto if the initialized selector is visible in the DOM.
