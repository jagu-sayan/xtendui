Toggle
Core

Toggle is the main js component that toggle classes, it's inherited by many other js components.

Components
Option Js

Setup

To use this component import the less and js files accordingly:

@import 'xtend-library/src/core/toggle/toggle.less';

import '/src/core/toggle/toggle.js'

Or just import core:

import '/src/xtend-core.js'

To modify a less or js file add webpack resolve and fork the file copying it in your project.

Usage

Self

You can use this markup to create a toggle with no targets.

Multiple

You can create a toggle with mutiple mode by assigning the toggle to a container element, elements and targets must be inside the container.

With multiple mode, the targets activated are the ones with the same index of the toggled element.

Use :scope when using descendant selector on root (e.g. :scope > *).

SyntaxDefaultDescription
Optionelements:Query:scope > a, :scope > buttonElements that triggers the events
Optiontargets:Query:scope > [class^="toggle-"], :scope > [class*=" toggle-"]Targets that gets activated on events by the elements
OptionelementsInner:QueryfalseAdditional query inside elements that gets activated on events
OptiontargetsInner:QueryfalseAdditional query inside targets that gets activated on events

Unique

The unique mode is useful when triggering targets outside the scope of the toggle.

To activate unique mode you need to specify targets with #id.

Initialization

Initialize automatically within markup with [data-xt-toggle="{ <options> }"] on the object:

Object is the DOM element you want to assign the component.

Or initialize with javascript:

let self = new Xt.Toggle(document.querySelector('#my-object'), {
  // options
})

Or inizialize with mutation observer (preferred method):

Xt.mount.push({
  matches: '#my-object',
  mount: object => {
    // init

    let self = new Xt.Toggle(object, {
      // options
    })

    // unmount

    const unmount = () => {
      self.destroy()
      self = null
    }
    return unmount
  }
})
