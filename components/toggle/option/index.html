Option Toggle
Core

Toggle

Min and Max

You can specify min and max concurrent activations. The min option is fulfilled on initialization, the max options deactivates the first activated when max is reached.

SyntaxDefaultDescription
Optionmin: Number0Minimum number of active elements
Optionmax: Number1Maximum number of active elements

Class

You can specify classes to toggle with class: 'my-class-0 my-class-1'. Toggled class is by default .active.

SyntaxDefaultDescription
Optionclass: QuerySelector'active'Specify activation class
OptionclassIn: QuerySelector'in'Specify in class
OptionclassOut: QuerySelector'out'Specify out class
OptionclassInitial: QuerySelector'initial'Specify initial class
OptionclassInverse: QuerySelector'inverse'Specify inverse class
You can start with toggled elements and targets just by adding the first class to the elements or targets. Following this activation the min option will take effect.

Event

You can specify on and off events for the toggle.

SyntaxDefaultDescription
Optionon: String'click'Set event to listen for activation
Optionoff: StringfalseSet event to listen for deactivation if not present options.on deactivate when already activated
If you set "on": "mouseenter" or "on": "mousehover", when the element have links on the tag or inside, they automatically get triggered only on double click when using touch devices.

Group

If you need to toggle multiple targets with the same element assign data-xt-group to elements and the associated targets targets. Group's additional elements aren't counted for min and max.

Auto

Auto doesn't run if the toggle is display: none.

SyntaxDefaultDescription
Optionauto: { time: Milliseconds }falseSet auto timing
Optionauto: { step: Number }1Set auto step
Optionauto: { initial: Boolean }trueRun auto on initialization (only if toggle is visible and one already active)
Optionauto: { pause: QuerySelector }falseQuery internal elements to puase on mouseenter and resume on mouseleave
Optionauto: { inverse: Boolean }falseSet auto inverse order
Use auto events.

Jump

Use jump: true to enable clicking on a target to jump to them.

SyntaxDefaultDescription
Optionjump: BooleanfalseEnable clicking on targets to jump to them

Navigation

You can add navigation with navigation: QuerySelector, set the amount to add (+1) or remove (-1) to the current activation index with [data-xt-nav="value"].

SyntaxDefaultDescription
Optionnavigation: QuerySelectorfalseNavigation with [data-xt-nav="value"]

Loop

Use loop: false to disable looping on activation.

SyntaxDefaultDescription
Optionloop: BooleantrueLoop on activation (auto, navigation, etc..)

Timing

Activation/deactivation automatically waits for the duration of animations and transitions on .active and .out.

You can use a function for delayOn and delayOff for example function(current, total) {return Math.min((total - current) * 150, 300)}.

SyntaxDefaultDescription
OptiondurationOn: MillisecondsfalseSet activation duration
OptiondurationOff: MillisecondsfalseSet deactivation duration
OptiondelayOn: Milliseconds|FunctionfalseSet activation delay
OptiondelayOff: Milliseconds|FunctionfalseSet deactivation delay
Optioninstant: Boolean|Object{ elements: true, targets: false, elementsInner: true, targetsInner: false }Set instant activation and deactivation

Keyboard

Use keyboard: { selector: 'object' } or keyboard: { selector : QuerySelector } to enable keyboard navigation on focus.

Use aria: false to disable aria generation, or granularly see @TODO.
